# CrowdDev 众开协商引擎 — 用户使用说明书

## 访问地址

**https://crowddev-production.up.railway.app**

---

## 一、系统概述

CrowdDev 是一个 AI 多角色协商设计工具。系统通过模拟多个 AI 角色（如警务人员、技术人员等），让它们围绕产品设计议题进行结构化协商，暴露冲突并达成共识，最终驱动 PRD 文档的迭代优化。

**核心流程：** 创建角色 → 发起协商会话 → 四步协商（表态→分析→辩论→共识）→ PRD 更新

## 三、角色管理

### 3.1 预置角色

系统预装了 3 个 CityMatrix 城市安防场景角色：

| 角色 | 职位 | 说明 |
|------|------|------|
| 👮 李Sir | 调度员 | 关注指挥效率和操作便捷 |
| 🚔 张警官 | 巡逻警员 | 关注现场执行和实用性 |
| 🔧 王工 | 系统管理员 | 关注系统稳定和技术可行性 |

### 3.2 新建角色

1. 进入「角色管理」页面
2. 点击右上角「新建角色」按钮
3. 填写角色信息：
   - **角色名称**：如"陈队长"
   - **头像 Emoji**：如 🧑‍✈️
   - **职位**：如"刑侦队长"
   - **组织**：如"市公安局刑侦支队"
   - **职责**（每行一条）：描述该角色的工作职责
   - **决策权限**（每行一条）：该角色可以做出的决策
   - **专业背景**（每行一条）：专业领域和经验
   - **性格特征**（每行一条）：如"谨慎务实"、"重视证据"
   - **核心关切**（每行一条）：该角色最关心的问题
4. 点击「保存」

### 3.3 编辑与版本管理

- 点击角色卡片上的 ✏️ 编辑按钮修改角色
- 系统自动进行**语义版本管理**：
  - 修改名称/职位/头像 → 补丁版本 +1（如 1.0.0 → 1.0.1）
  - 修改核心关切/性格/专业/决策权 → 次版本 +1（如 1.0.0 → 1.1.0）
- 点击 🕐 按钮查看角色的完整版本历史

### 3.4 删除角色

点击角色卡片上的 🗑️ 按钮，确认后删除（软删除，不影响历史数据）。

## 四、协商会话（核心功能）

### 4.1 创建会话

1. 进入「协商会话」页面
2. 点击左侧列表上方的 ➕ 按钮
3. 填写会话信息：
   - **协商议题**：如"视频追踪功能是否需要二次确认？"
   - **议题描述**（可选）：补充说明
   - **协商阶段**：选择 设计期 / 验收期 / 运营期
   - **参与角色**：点击选择参与协商的角色（至少选1个）
4. 点击「创建会话」

### 4.2 四步协商流程

创建会话后，在右侧操作栏按顺序点击按钮执行协商步骤：

#### 步骤一：启动表态 ▶️

- 每个参与角色从自身立场出发，发表对议题的看法
- 系统会实时显示"XX 正在表态..."的思考动画
- 所有角色表态完毕后，进入下一步

#### 步骤二：分析冲突 🔍

- AI 分析所有表态中的核心冲突点
- 输出冲突摘要、涉及角色、严重程度、引导问题
- 结果以紫色「冲突分析」标签显示

#### 步骤三：辩论回应 ⚡

- 各角色基于冲突分析结果进行回应
- 可在「引导问题」输入框中输入主持人问题（可选），引导讨论方向
- 角色可以坚持立场、做出妥协、或提出新方案
- 可多次运行辩论回合（分析完成后或辩论完成后均可再次辩论）

#### 步骤四：寻求共识 🤝

- AI 综合所有讨论记录生成共识方案
- 不同阶段生成不同格式的共识：
  - **设计期**：设计决策 + 妥协方案 + 约束条件 + PRD 修改建议
  - **验收期**：通过/有条件通过/不通过 + 偏差项 + 改进项
  - **运营期**：运营洞察 + 优化建议 + 参数调整建议

#### 步骤五：PRD检查 📋（可选）

- 基于共识结果，检查是否还有未解决冲突
- 生成具体的 PRD 修改条目（增/改/删）

### 4.3 界面说明

- **顶部进度条**：显示协商进展（表态 → 分析 → 辩论 → 共识 → PRD），当前步骤高亮
- **消息颜色标识**：
  - 🔵 蓝色 = 立场表态
  - 🟣 紫色 = 冲突分析
  - 🟡 琥珀色 = 辩论回应
  - 🟢 绿色 = 共识方案 / PRD更新
- **消息内容**：支持 Markdown 格式渲染（标题、列表、加粗等）
- **消息时间戳**：每条消息右上角显示发送时间
- **删除会话**：鼠标悬停在左侧会话列表项上，右上角出现删除按钮

### 4.4 状态流转说明

```
已创建 → 表态中 → 表态完成 → 分析中 → 分析完成
                                         ↓
                              ┌── 辩论中 → 辩论完成 ──┐
                              │          ↑             │
                              │          └─────────────┘ (可多轮辩论)
                              ↓
                         生成共识 → 已达成共识 → PRD检查中 → PRD已更新
```

## 五、PRD 智能拆解

将项目文档自动拆解为可协商的议题。

### 5.1 操作步骤

**第一步：选择文档**

1. 点击「加载文档列表」获取 project-docs 目录下的 Markdown 文档
2. 点击选择要拆解的文档
3. 点击「开始预筛」

**第二步：预筛章节**

- AI 对文档各章节评分（1-10 分），高分表示更值得协商
- 评分标准：
  - 9-10分：涉及多角色利益冲突的设计决策
  - 7-8分：涉及不同角色工作流程差异的设计
  - 5-6分：有设计内容但不太涉及角色差异
  - 3-4分：纯技术实现细节
  - 1-2分：目录、版本号等无实质内容
- 每个章节显示：评分、协商价值（高/中/低）、评分理由、潜在冲突提示
- 可使用快捷按钮：「选高价值」（≥7分）/ 「全选」/ 「清空」
- 选择需要的章节后，点击「提取议题」

**第三步：提取议题**

- AI 从选中章节提取具体协商议题
- 每个议题显示：议题名称、描述、涉及角色、优先级（高/中/低）
- 选择议题后点击「批量创建会话」，自动创建对应的协商会话

### 5.2 缓存机制

- 预筛结果会自动缓存，相同文件再次预筛时直接返回缓存结果
- 文档内容修改后，缓存自动失效，重新调用 AI 评分
- 如需强制刷新，可添加 `?cache=false` 参数

## 八、典型使用流程

```
1. 创建/确认角色 → 确保有 2-3 个不同立场的角色
        ↓
2. (可选) PRD 拆解 → 从文档自动提取议题
        ↓
3. 创建协商会话 → 设定议题、选择阶段和参与角色
        ↓
4. 执行四步协商 → 表态 → 分析 → 辩论（可多轮）→ 共识
        ↓
5. PRD 检查 → 生成文档修改建议
        ↓
6. 查看演化追踪 → 监控整体进展和数据
```

## 九、注意事项

1. **角色数量**：建议每次协商选择 2-4 个角色，角色间应有不同立场和关切
2. **协商阶段**：不同阶段（设计期/验收期/运营期）的 AI 引导策略不同，请根据实际情况选择
3. **辩论轮次**：可多次运行辩论，让角色充分讨论后再求共识
4. **引导问题**：辩论时输入引导问题可以聚焦讨论方向，提高协商效率
5. **AI 响应时间**：每步协商需要调用 Claude API，通常需要 5-15 秒等待
6. **数据持久化**：所有数据存储在 SQLite 数据库中，服务重启后数据保留
7. **文档格式**：PRD 拆解仅支持 Markdown (.md) 格式的文档
8. **并发限制**：同一会话请勿同时运行多个协商步骤，需等待当前步骤完成
